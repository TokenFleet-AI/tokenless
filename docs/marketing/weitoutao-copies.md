## 二、文案系列（5 种类型轮换）

### 类型 1：日常工作吐槽型（推荐首发）

> 团队上 Claude Code 之后，token 消耗比预期高了 3 倍。查了一下，大部分不是写代码花的，是 Claude 执行 git、cargo 这些命令时，吃掉了大量无用的终端输出。
>
> 当技术总监的，钱不能这么烧。花了两个周末写了个 Rust 小工具，自动过滤这些噪音，token 消耗降了 60% 多。
>
> 顺手开源了，GitHub 搜 TokenFleet-AI / tokenless
>
> #初创公司 #AI编程 #Rust #皮哥IT日常

**配图建议**：Claude Code 终端截图，token 使用量对比。

### 类型 2：数据对比型

> 同一个 JSON 响应，不同格式消耗的 token：
>
> - 原始 JSON：~2800 tokens
> - 压缩后 JSON：~1200 tokens（-57%）
> - TOON 编码格式：~680 tokens（-76%）
>
> 实测数据。做了 4 件事：
> 1. Schema 压缩（工具定义减 57%）
> 2. 响应压缩（去空值、截断长字符串）
> 3. TOON 编码（JSON → 类 YAML 紧凑格式）
> 4. 命令重写（git/ls/cargo 等 70+ 命令输出过滤）
>
> 纯 Rust，零运行时依赖，单文件二进制。
> GitHub 搜 TokenFleet-AI / tokenless

**配图建议**：token 节省对比柱状图。

---

### 类型 3：技术分享型

> 皮哥最近研究了个有意思的东西——TOON 编码。
>
> 简单说就是把 JSON 用更紧凑的格式表示，同样的数据给到 LLM，token 消耗直接少一大半。
>
> 比如 `{"name":"Alice","age":30,"role":"admin"}` 变成：
> ```
> name: Alice
> age: 30
> role: admin
> ```
>
> 看着简单，但类型推断、嵌套结构处理、可逆编解码，用 Rust 实现起来还是有点意思的。
>
> 完整代码开源了，GitHub 搜 TokenFleet-AI / tokenless
>
> #Rust学习 #技术分享 #LLM

**配图建议**：代码片段截图或 TOON 格式效果截图。

### 类型 4：踩坑记录型

> 踩了个坑：Claude Code 执行 `git status` 时，如果项目里有 `.env` 文件，输出里会包含敏感信息，这些都被 Claude 吃进去了。
>
> 两个问题：1）浪费 token；2）安全隐患。
>
> 写了个工具在命令输出到达 Claude 之前做一层过滤，既省 token 又防泄露。
>
> 全栈民工的日常：发现问题 → 解决问题 → 开源分享。
> GitHub 搜 TokenFleet-AI / tokenless
>
> #安全 #AI编程 #踩坑记录

**配图建议**：终端输出中泄露 `.env` 的截图（打码后）。
