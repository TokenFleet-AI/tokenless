# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/),

## [0.3.0] - 2026-05-29

### Added
- Interactive TUI dashboard (`tokenless tui`) with ratatui
  - Dashboard tab with stat cards, breakdown bars, recent activity
  - Records tab with scrollable table, detail view
  - Agents tab with per-agent breakdown and drill-down
  - Trends tab with Sparkline charts (daily chars/tokens saved)
  - Search (`/` key), time range filter (`t` key)
  - JSON export (`e` key)
  - Config panel (`c` key), help overlay (`?` key)
- Chinese/English bilingual support (default Chinese, `--lang en`)
- CI/CD pipeline (.github/workflows/ci.yml, cd.yml, release.yml)
- Performance benchmark report (docs/performance-benchmarks.md)

### Changed
- Version 0.2.0 → 0.3.0
- `main.rs` refactored: command handlers extracted to `handlers.rs` (from ~1000 to ~280 lines)
- `env_check.rs` refactored into directory module (`spec.rs`, `checker.rs`, `fixer.rs`)
- Shared utilities extracted to `util.rs` (DB path, recorder, cmd checking)
- Cache implementation: Vec-based LRU replaced with `lru` crate (O(1) operations)
- `TOKENLESS_CACHE_SIZE` env var now actually read and respected
- Cache skips hashing for inputs under 64 bytes
- CLI commands eliminate double `to_string` + `to_string_pretty` serialization
- MCP server uses static `LazyLock<Compressor>` instances
- `McpRequest` now uses `#[serde(deny_unknown_fields)]`
- Stats recording now sanitizes sensitive text (Bearer tokens, API keys)
- `truncate_description()` reduces intermediate String allocations from 5 to 2

### Fixed
- MCP server input size limit (10MB, DoS protection)
- Stack overflow in deeply nested schema compression (depth guard)
- Double BOM stripping for Cursor on Windows
- Pre-existing clippy warnings (unreachable patterns, disallowed methods)
- Color scheme for dark terminal backgrounds (DarkGray → Gray)

## [0.2.0] - 2025-05-20

### Added
- MCP JSON-RPC 2.0 server over stdin/stdout (7 tools)
- Intelligent format router with 3 encoding strategies
- Differential response compression for repeated tool calls
- OpenClaw and Hermes agent plugin adapters
- `init` command for agent hook installation (11 agents supported)
- Comprehensive user guide (docs/user-guide.md)
- 14 design specification documents (specs/0001-0014)

### Changed
- Predictive cache now uses blake3 hashing with configurable size
- Improved CJK/Unicode handling in string truncation
- Enhanced error reporting with exit code semantics
- Stats system supports disable/enable toggle

### Fixed
- Stack overflow in deeply nested schema compression (depth guard)
- Double BOM stripping for Cursor on Windows
- Enum truncation marker wording for clarity
- PascalCase field detection in response compression
- RTK not-found fallback messaging
